📹 TasteLab - Data Integration Project

An 18-week specialization internship focused on full-stack development and system integration—building backend data pipelines that process multi-camera video data through transcription, NLP analysis, and computer vision models.

🔄 Dagster 🐳 Docker 📦 MinIO 🎙️ NeMo/Whisper 🐍 Python 🗄️ PostgreSQL

Project Overview

TasteLab is a research initiative at Breda University focused on analyzing food experiments through video data. My role as a supporting intern involves building the core infrastructure that enables researchers to process and analyze multi-camera recordings of cooking experiments.

This is a collaborative project where I work alongside teammates specializing in computer vision (Joey), NLP (Ayumi), and other areas (Jeremy), with mentorship from Arash. My focus is on the full-stack integration layer—connecting all the ML components into a cohesive, automated pipeline.

Technical Architecture

The system I'm building processes video data through multiple stages:

  • Data Ingestion: Automated file monitoring and ingestion from MinIO object storage, handling multi-camera video uploads with proper metadata extraction.
  • Audio Processing: Extracting audio tracks from video files and preprocessing for transcription.
  • Transcription Pipeline: Implementing speech-to-text using NVIDIA NeMo toolkit and OpenAI Whisper models, with speaker diarization support.
  • NLP Analysis: Processing transcripts through NLP models for summarization, key moment extraction, and content structuring.
  • Computer Vision Integration: Connecting to CV models for visual analysis of cooking activities and ingredient detection.
  • Data Storage: Structured storage of all outputs in PostgreSQL with proper relationships and MinIO for binary artifacts.

Dagster Orchestration

The entire pipeline is orchestrated using Dagster, a modern data orchestration framework:

  • Defined assets for each processing stage with proper dependencies
  • Implemented sensors for automated triggering on new file uploads
  • Built custom I/O managers for MinIO and PostgreSQL integration
  • Created partitioned assets for efficient batch processing
  • Set up monitoring and alerting for pipeline health

Infrastructure Challenges

Working on shared university servers presented unique challenges:

  • GPU Resource Management: Implementing proper CUDA memory allocation to share GPUs with other processes and users.
  • Docker Containerization: Creating optimized containers that balance image size with functionality, managing complex ML dependencies.
  • Dependency Conflicts: Resolving conflicts between computer vision packages (OpenCV, PyTorch) and NLP libraries (NeMo, Transformers).
  • Network Configuration: Setting up proper communication between containerized services and external storage.

Major milestones achieved during this internship:

Technologies Deep Dive

Dagster — Data orchestration framework for building, running, and monitoring data pipelines with software-defined assets.
MinIO — High-performance object storage compatible with Amazon S3 API, used for storing video files and processed artifacts.
NVIDIA NeMo — Toolkit for building conversational AI models, used for speech recognition and transcription.
Docker — Containerization platform ensuring reproducible environments across development and production.
PostgreSQL — Relational database storing structured metadata, transcripts, and analysis results.
